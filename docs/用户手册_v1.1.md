## 1. 产品概述

PLC 数据监控系统（PLCMonitor v1.1）是一款基于 Qt 6 开发的跨平台工业自动化监控软件。系统通过多协议通讯架构，支持多种主流 PLC 通讯协议，实现对现场设备数据的实时采集、可视化展示、历史存储与分析。

### 1.1 核心能力

| 能力 | 说明 |
|------|------|
| **多协议通讯** | 支持西门子 S7（Snap7）、Modbus TCP（libmodbus）、Modbus RTU（libmodbus）三种协议 |
| **多设备管理** | CommManager 统一管理多台 PLC/设备，每台设备独立线程运行 |
| **实时数据采集** | 按设备独立配置轮询间隔，支持批量读取优化 |
| **多视图展示** | 工艺流程图、实时数据表、趋势图表、历史趋势四种视图 |
| **双数据库策略** | MySQL 主数据库 + SQLite 本地备库，断网时自动降级、恢复后自动同步 |
| **权限管控** | 管理员/工程师/操作员/观察者四级 RBAC 权限体系 |
| **报警管理** | 支持阈值报警配置，多级别告警（信息/警告/错误/严重） |
| **数据导出** | 支持实时数据和历史数据导出为 CSV 格式 |

### 1.2 支持的设备与协议

| 协议 | 适用设备 | 通讯库 |
|------|----------|--------|
| S7 (ISO-on-TCP) | 西门子 S7-200/300/400/1200/1500 | Snap7 v1.4.0 |
| Modbus TCP | 变频器、仪表、PLC 等通过以太网的 Modbus 设备 | libmodbus 3.1.12 |
| Modbus RTU | 电表、传感器、小型 PLC 等通过串口的 Modbus 设备 | libmodbus 3.1.12 |

### 1.3 通讯架构

```
Application（应用层）
  └── CommManager（通讯管理器）
        ├── S7Connector      → 西门子 S7 PLC（Snap7 TCP/ISO）
        ├── ModbusTcpConnector → Modbus TCP 设备（libmodbus）
        └── ModbusRtuConnector → Modbus RTU 设备（libmodbus 串口）
              ↓
        PLCData（统一数据模型，tagKey 格式："设备ID/标签名"）
```

每台设备在独立 QThread 中运行，互不影响。CommManager 统一管理连接生命周期和轮询调度。

## 3. 安装与部署

### 3.1 Windows 安装

1. 解压发布包到目标目录（如 `D:\PLCMonitor`）
2. 确保 `snap7.dll` 位于程序目录中（用于 S7 通讯）
3. 运行 `PLCMonitor.exe`

### 3.2 Linux (Ubuntu) 安装

```bash
# 安装依赖
sudo apt update
sudo apt install libsnap7 qt6-base-dev

# 解压并运行
tar -xzf plcmonitor-linux.tar.gz -C /opt/plcmonitor
cd /opt/plcmonitor
./PLCMonitor
```

### 3.3 目录结构

```
安装目录/
├── PLCMonitor(.exe)        # 主程序
├── snap7.dll/.so           # Snap7 S7 通讯库（libmodbus 已静态编译）
├── configs/
│   └── devices.json        # 设备与标签配置文件
├── resources/
│   └── styles/             # 主题样式文件（mac.css / dark.css / light.css）
└── logs/
    └── plcmonitor.log       # 运行日志
```

## 4. 首次启动与配置

### 4.1 启动程序

首次启动时，程序会自动创建 `configs/devices.json` 配置文件。默认含有一台示例 S7 设备配置。

### 4.2 设备配置文件格式

配置文件 `configs/devices.json` 是整个系统的核心配置，包含所有设备的连接参数和标签定义。

#### S7 设备示例

```json
{
  "devices": [
    {
      "id": "PLC_S7_01",
      "name": "主控 PLC",
      "protocol": "S7",
      "ip": "192.168.0.11",
      "rack": 0,
      "slot": 1,
      "port": 102,
      "pollingInterval": 1000,
      "tagGroups": [
        {
          "name": "电机状态",
          "tags": [
            { "name": "运行状态", "address": "DB1.DBX0.0", "dataType": "BOOL", "writable": false },
            { "name": "运行频率", "address": "DB1.DBW2",   "dataType": "REAL", "writable": true }
          ]
        }
      ]
    }
  ]
}
```

#### Modbus TCP 设备示例

```json
{
  "id": "INV_01",
  "name": "1# 变频器",
  "protocol": "ModbusTCP",
  "ip": "192.168.1.200",
  "port": 502,
  "modbusSlaveId": 1,
  "pollingInterval": 500,
  "tagGroups": [
    {
      "name": "运行参数",
      "tags": [
        { "name": "频率设定", "address": "3:100", "dataType": "REAL",  "writable": true },
        { "name": "运行电流", "address": "4:200", "dataType": "REAL",  "writable": false },
        { "name": "运行状态", "address": "1:0",   "dataType": "BOOL",  "writable": false }
      ]
    }
  ]
}
```

#### Modbus RTU 设备示例

```json
{
  "id": "METER_01",
  "name": "1# 电表",
  "protocol": "ModbusRTU",
  "serialPort": "COM3",
  "baudRate": 9600,
  "dataBits": 8,
  "stopBits": 1,
  "parity": "N",
  "modbusSlaveId": 2,
  "pollingInterval": 2000,
  "tagGroups": [
    {
      "name": "电气参数",
      "tags": [
        { "name": "电压", "address": "4:0", "dataType": "REAL", "writable": false },
        { "name": "电流", "address": "4:2", "dataType": "REAL", "writable": false }
      ]
    }
  ]
}
```

### 4.3 设备配置字段说明

| 字段 | 类型 | 说明 | S7 | ModbusTCP | ModbusRTU |
|------|------|------|:--:|:---------:|:---------:|
| `id` | 字符串 | 设备唯一 ID | ✅ | ✅ | ✅ |
| `name` | 字符串 | 设备显示名称 | ✅ | ✅ | ✅ |
| `protocol` | 字符串 | 协议类型："S7"/"ModbusTCP"/"ModbusRTU" | ✅ | ✅ | ✅ |
| `ip` | 字符串 | IP 地址 | ✅ | ✅ | — |
| `port` | 整数 | 端口号（S7 默认 102，Modbus 默认 502）| ✅ | ✅ | — |
| `rack` | 整数 | 机架号（S7-300/400 用） | ✅ | — | — |
| `slot` | 整数 | 插槽号（S7-1200/1500 填 1） | ✅ | — | — |
| `modbusSlaveId` | 整数 | Modbus 从站地址（1-247） | — | ✅ | ✅ |
| `serialPort` | 字符串 | 串口名称（如 COM3、/dev/ttyUSB0） | — | — | ✅ |
| `baudRate` | 整数 | 波特率（9600/19200/38400/57600/115200） | — | — | ✅ |
| `dataBits` | 整数 | 数据位（7 或 8） | — | — | ✅ |
| `stopBits` | 整数 | 停止位（1 或 2） | — | — | ✅ |
| `parity` | 字符串 | 校验位（N=无, E=偶, O=奇） | — | — | ✅ |
| `pollingInterval` | 整数 | 轮询间隔（毫秒，100-10000） | ✅ | ✅ | ✅ |

### 4.4 标签地址格式

不同协议使用不同的地址格式：

| 协议 | 地址格式 | 示例 | 说明 |
|------|----------|------|------|
| **S7** | `DB<编号>.DB<类型><偏移>[.<位>]` | `DB1.DBX0.0` | DB=数据块, X=BOOL, B=BYTE, W=WORD, D=DWORD |
| **Modbus** | `<功能码>:<寄存器地址>` | `3:100` | 功能码 1=线圈, 2=离散输入, 3=保持寄存器, 4=输入寄存器 |

**数据类型说明**:

| 类型 | 字节数 | 说明 | S7 | Modbus |
|------|--------|------|:--:|:------:|
| BOOL | 1 bit | 布尔值，Modbus 对应 1 个线圈或 1 个寄存器的最低位 | ✅ | ✅ |
| BYTE | 1 | 8 位无符号整数 | ✅ | — |
| WORD | 2 | 16 位无符号整数（Modbus 对应 1 个寄存器） | ✅ | ✅ |
| DWORD | 4 | 32 位无符号整数（Modbus 对应 2 个连续寄存器，大端） | ✅ | ✅ |
| INT | 2 | 16 位有符号整数 | ✅ | ✅ |
| DINT | 4 | 32 位有符号整数 | ✅ | ✅ |
| REAL | 4 | IEEE 754 单精度浮点数 | ✅ | ✅ |
| STRING | 可变 | 字符串 | ✅ | — |

> **注意**：Modbus 多寄存器类型（DWORD/DINT/REAL）按大端字节序解析，即高16位在前。若您的设备使用小端序，需调整配置。

## 5. 通讯协议与设备管理

### 5.1 通讯管理器（CommManager）

v1.1 引入通讯管理器统一管理多设备多协议。工作流程：

1. 读取 `devices.json` 中每个设备的 `protocol` 字段
2. 根据协议类型创建对应的连接器（S7Connector / ModbusTcpConnector / ModbusRtuConnector）
3. 每台设备分配独立工作线程
4. 按各设备的 `pollingInterval` 独立轮询
5. 断线时自动重连（可配置）

### 5.2 多设备共存

系统支持同时连接多台不同协议设备：

```
示例场景：
├── PLC_S7_01     (S7, 192.168.1.100)     — 主控 PLC
├── INV_01        (ModbusTCP, 192.168.1.200)  — 变频器
├── INV_02        (ModbusTCP, 192.168.1.201)  — 变频器
└── METER_01      (ModbusRTU, COM3)       — 电表
```

各设备完全独立：一台断线不影响其他设备的数据采集。

### 5.3 设备配置界面

在菜单栏 **设备 → 设备与标签配置** 中打开设备配置对话框：

1. 点击 **添加设备**，输入设备 ID 和名称
2. 在属性面板的 **协议** 下拉框中选择对应协议
3. 根据协议类型填写连接参数
4. 添加标签分组和标签
5. 点击 **保存配置** 持久化修改

## 7. 功能模块详解

### 7.1 PLC 连接管理

#### 手动连接
1. 点击顶部工具栏的 **"连接PLC"** 按钮
2. 系统根据 `devices.json` 中各设备的协议和参数自动连接
3. 连接成功后状态栏显示 **"设备ID: ●已连接"**

#### 自动重连
每台设备独立维护重连策略（默认 3 秒重试一次，最多 3 次）。

#### 状态指示灯

| 状态 | 颜色 | 说明 |
|------|------|------|
| 已连接 | 绿色 | 设备连接正常，数据正在采集 |
| 连接中 | 黄色 | 正在尝试连接设备 |
| 已断开 | 红色 | 设备连接已断开（不影响其他设备） |
| 轮询中 | 蓝色 | 正在周期性读取设备数据 |

### 7.6 标签管理器

标签管理器位于右侧面板，用于选择当前视图要监控的标签。

- **复选框**：勾选/取消勾选标签
- **全选**：选中当前分组所有标签
- **清除**：取消所有选择
- **应用**：将选择应用到当前视图

标签按 **设备 → 分组 → 标签** 的三级树形结构组织。

### 7.7 控制面板

用于批量写入 PLC/设备数据。

1. 左侧导航点击 **"控制面板"**
2. 输入目标标签和写入值

### 7.8 报警中心

### 7.9 数据写入

## 9. 数据库配置

### 9.1 双数据库策略

```
MySQL（主数据库）
  ├── 存储历史数据、报警记录
  └── 连接断开时自动切换到 SQLite

SQLite（本地备库）
  ├── 断网应急存储
  └── MySQL 恢复后自动同步数据
```

### 9.2 配置方法

在 `devices.json` 的根节点配置：

```json
{
  "database": {
    "mysql": {
      "host": "localhost",
      "port": 3306,
      "database": "plcmonitor",
      "user": "root",
      "password": ""
    }
  }
}
```

## 11. 常见问题与故障排除

### 11.1 连接问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| S7 连接失败 | IP/Rack/Slot 配置错误 | 检查 PLC 实际参数，S7-1200/1500 需启用 PUT/GET 访问 |
| S7 连接超时 | 防火墙阻止端口 102 | 添加防火墙入站规则允许 TCP 102 |
| ModbusTCP 连接失败 | IP/端口配置错误 | 默认端口 502，检查从站地址是否正确 |
| ModbusTCP 超时 | 网络延迟 | 增加轮询间隔或响应超时 |
| ModbusRTU 连接失败 | 串口被占用或权限不足 | Linux 需将用户加入 dialout 组，Windows 检查 COM 端口号 |
| 多设备场景某台断线 | 不影响其他设备 | 查看状态栏确认断线设备 ID，单独排查 |

### 11.2 Modbus 常见错误码

| 错误 | 说明 | 排查方向 |
|------|------|----------|
| `Connection refused` | 目标端口未开放 | 检查设备 IP 和端口 |
| `Illegal data address` | 寄存器地址超出范围 | 查阅设备手册确认寄存器范围 |
| `Illegal function` | 功能码不支持 | 确认该寄存器是否支持读/写 |
| `Slave device failure` | 从站设备错误 | 检查设备是否正常运行 |

### 11.3 数据展示问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 数据显示为 0 | 标签未被选中 | 在标签管理器中勾选并点击应用 |
| 浮点数显示异常 | Modbus 字节序不一致 | 大端/小端问题，检查设备手册 |
| 报警不触发 | 报警阈值未配置 | 在标签属性中设置报警上限/下限 |

### 11.4 日志排查

程序运行日志位于 `<安装目录>/logs/plcmonitor.log`，包含：
- 设备连接/断开记录
- 通讯错误详情
- 数据库操作日志
