# 通讯管理器重构方案

> 版本：v1.2 | 日期：2026-06-13  
> 代码目录：`D:/plcmonitor20260610`  
> 基于当前重构后的 `PLCData / DeviceConfig / tagKey` 数据架构

---

## 一、重构背景与目标

### 1.1 现状问题

| 问题 | 说明 |
|------|------|
| **单设备绑定** | `PLCConnector` 持有一个 `S7Object`，只能连接一台 PLC |
| **协议耦合** | `PLCConnector` 直接调用 Snap7 API，无法扩展 Modbus TCP/RTU 等协议 |
| **Controller 强依赖** | `Controller` 直接持有 `PLCConnector*`，多设备需要大量 if/else |
| **线程管理分散** | 每个连接器的线程生命周期由 `Application` 手动管理 |
| **通讯代码散落 core** | 通讯相关类混在业务逻辑 `core` 中，职责不清 |

### 1.2 重构目标

1. **新建 `comm` 目录**：所有通讯相关类独立成库，与 `core`、`app` 平级
2. **多设备支持**：同一协议多台设备可独立连接、独立轮询
3. **多协议扩展**：`IConnector` 抽象接口，支持 S7（Snap7）、Modbus TCP/RTU（libmodbus）
4. **统一通讯入口**：`Controller` 只与 `CommManager` 通信，不感知具体协议
5. **集中线程管理**：`CommManager` 统一管理所有连接器的线程、重连、心跳

### 3.2 `cmake/FindLibModbus.cmake`（新建）

```cmake
# cmake/FindLibModbus.cmake
if(TARGET LibModbus::LibModbus)
    return()
endif()

if(WIN32)
    set(LIBMODBUS_ROOT "${CMAKE_SOURCE_DIR}/third_party/libmodbus")
    set(LIBMODBUS_INCLUDE_DIR "${LIBMODBUS_ROOT}/include")

    find_library(LIBMODBUS_LIBRARY
        NAMES modbus libmodbus
        PATHS "${LIBMODBUS_ROOT}/lib"
        NO_DEFAULT_PATH
    )

    # 查找 DLL（供运行时拷贝）
    find_file(LIBMODBUS_DLL
        NAMES modbus.dll libmodbus.dll
        PATHS "${LIBMODBUS_ROOT}/lib" "${LIBMODBUS_ROOT}/bin"
        NO_DEFAULT_PATH
    )

    if(NOT LIBMODBUS_INCLUDE_DIR OR NOT LIBMODBUS_LIBRARY)
        message(FATAL_ERROR
            "libmodbus not found in ${LIBMODBUS_ROOT}.\n"
            "请将 libmodbus 预编译库放入 third_party/libmodbus/，\n"
            "或参考方案文档中的获取方式。")
    endif()

    add_library(LibModbus::LibModbus IMPORTED SHARED)
    set_target_properties(LibModbus::LibModbus PROPERTIES
        IMPORTED_IMPLIB "${LIBMODBUS_LIBRARY}"
        INTERFACE_INCLUDE_DIRECTORIES "${LIBMODBUS_INCLUDE_DIR}"
    )
    if(LIBMODBUS_DLL)
        set_target_properties(LibModbus::LibModbus PROPERTIES
            IMPORTED_LOCATION "${LIBMODBUS_DLL}"
        )
    endif()

else()
    # Linux：通过 pkg-config 查找
    find_package(PkgConfig REQUIRED)
    pkg_check_modules(LIBMODBUS REQUIRED libmodbus)

    add_library(LibModbus::LibModbus INTERFACE IMPORTED)
    set_target_properties(LibModbus::LibModbus PROPERTIES
        INTERFACE_INCLUDE_DIRECTORIES "${LIBMODBUS_INCLUDE_DIRS}"
        INTERFACE_LINK_LIBRARIES      "${LIBMODBUS_LIBRARIES}"
    )
endif()

set(LibModbus_FOUND TRUE)
message(STATUS "Found libmodbus: ${LIBMODBUS_LIBRARY}")
```

## 四、`IConnector` — 通讯抽象接口

```cpp
// src/comm/iconnector.h
#ifndef ICONNECTOR_H
#define ICONNECTOR_H

#include <QObject>
#include <QVariant>
#include "common_types.h"

class IConnector : public QObject
{
    Q_OBJECT

public:
    struct ReadRequest {
        QString tagKey;    // 格式：deviceId/tagName
        TagInfo tagInfo;
    };

    struct ReadResult {
        QString tagKey;
        QVariant value;
        bool success = false;
        QString error;
    };

    explicit IConnector(const QString &deviceId, QObject *parent = nullptr)
        : QObject(parent), m_deviceId(deviceId) {}
    virtual ~IConnector() = default;

    // -------- 连接管理 --------
    virtual bool connectToDevice() = 0;
    virtual void disconnectFromDevice() = 0;
    virtual bool isConnected() const = 0;
    virtual PLCConnectionState connectionState() const = 0;

    // -------- 数据读写 --------
    virtual QList<ReadResult> readBatch(const QList<ReadRequest> &requests) = 0;
    virtual bool writeTag(const TagInfo &tag, const QVariant &value) = 0;

    // -------- 轮询控制 --------
    virtual void startPolling(int intervalMs = 1000) = 0;
    virtual void stopPolling() = 0;
    virtual bool isPolling() const = 0;

    // -------- 重连参数 --------
    virtual void setAutoReconnect(bool enabled, int maxAttempts = 3, int intervalMs = 3000) = 0;

    // -------- 标识 --------
    QString deviceId() const { return m_deviceId; }
    virtual QString protocolName() const = 0;   // "S7" / "ModbusTCP" / "ModbusRTU"

    virtual void initInThread() = 0;

signals:
    void connected(const QString &deviceId);
    void disconnected(const QString &deviceId);
    void connectionError(const QString &deviceId, const QString &error);
    void connectionStateChanged(const QString &deviceId, PLCConnectionState state);
    void tagValueUpdated(const QString &tagKey, const QVariant &value);
    void batchDataUpdated(const QMap<QString, QVariant> &data);
    void writeCompleted(const QString &tagKey, bool success, const QString &error);
    void errorOccurred(const QString &deviceId, const QString &error, AlertLevel severity);
    void statusMessage(const QString &deviceId, const QString &message);

protected:
    QString m_deviceId;
};

#endif // ICONNECTOR_H
```

## 六、`ModbusTcpConnector` — libmodbus TCP 实现

### 6.1 libmodbus API 说明

```c
// libmodbus 核心 API（同步，适合在工作线程中直接调用）
modbus_t *modbus_new_tcp(const char *ip, int port);   // 创建 TCP 上下文
int  modbus_connect(modbus_t *ctx);                    // 建立连接
void modbus_close(modbus_t *ctx);
void modbus_free(modbus_t *ctx);

// 读操作（返回实际读取数量，<0 表示失败）
int modbus_read_bits(modbus_t *ctx, int addr, int nb, uint8_t *dest);           // FC1
int modbus_read_input_bits(modbus_t *ctx, int addr, int nb, uint8_t *dest);     // FC2
int modbus_read_registers(modbus_t *ctx, int addr, int nb, uint16_t *dest);     // FC3
int modbus_read_input_registers(modbus_t *ctx, int addr, int nb, uint16_t *dest); // FC4

// 写操作
int modbus_write_bit(modbus_t *ctx, int coil_addr, int status);                 // FC5
int modbus_write_register(modbus_t *ctx, int reg_addr, int value);              // FC6
int modbus_write_bits(modbus_t *ctx, int addr, int nb, const uint8_t *data);    // FC15
int modbus_write_registers(modbus_t *ctx, int addr, int nb, const uint16_t *data); // FC16

// 设置超时
modbus_set_response_timeout(ctx, 0, 500000);  // 500ms
```

### 6.2 地址格式约定

`TagInfo::address` 采用 `"FC:寄存器地址"` 格式，其中 FC 为功能码数字：

| address | libmodbus 函数 | 数据方向 | 典型用途 |
|---------|---------------|---------|---------|
| `"1:100"` | `modbus_read_bits` | 只读 | 线圈读取 |
| `"2:200"` | `modbus_read_input_bits` | 只读 | 离散输入 |
| `"3:300"` | `modbus_read_registers` | 读写 | 保持寄存器（最常用） |
| `"4:400"` | `modbus_read_input_registers` | 只读 | 输入寄存器 |

写操作：`TagInfo::writable = true` 且 FC 为 1 或 3 时启用写入。

### 6.3 多寄存器类型映射

| TagDataType | 寄存器数量 | 解析方式 |
|-------------|----------|---------|
| BOOL | 1（线圈）或 1（寄存器位0） | `uint16_t & 0x01` 或直接 bool |
| WORD | 1 | `uint16_t` |
| DWORD | 2 | 大端合并：`(reg[0] << 16) | reg[1]` |
| INT | 1 | `(int16_t)uint16_t` |
| DINT | 2 | 大端合并转 `int32_t` |
| REAL | 2 | `memcpy` 到 `float`（IEEE 754） |

## 七、`ModbusRtuConnector` — libmodbus RTU 实现

### 7.1 libmodbus RTU API

```c
// RTU 上下文创建（串口参数）
modbus_t *modbus_new_rtu(
    const char *device,  // "COM3" 或 "/dev/ttyUSB0"
    int baud,            // 9600, 19200, 115200 ...
    char parity,         // 'N'=None, 'E'=Even, 'O'=Odd
    int data_bit,        // 7 or 8
    int stop_bit         // 1 or 2
);

// 设置从站地址（每次请求前设置，或使用 modbus_set_slave）
modbus_set_slave(ctx, slaveId);

// 其余 read/write API 与 TCP 完全相同
```

RTU 和 TCP 的业务逻辑（地址解析、寄存器解析、类型映射）**完全一致**，只有上下文创建和串口参数不同。

### 7.2 `DeviceConfig` 新增字段

```cpp
// src/common/common_types.h 中 DeviceConfig 结构体新增：
struct DeviceConfig {
```
