## 1. 产品概述

PLC 数据监控系统是一款基于 Qt 6 开发的跨平台工业自动化监控软件，专为西门子 S7 系列 PLC 设计。系统通过 Snap7 协议实现对 PLC 数据的实时采集、可视化展示、历史存储与分析，帮助工程师和操作人员高效监控生产现场设备运行状态。

### 核心能力

| 能力 | 说明 |
|------|------|
| **实时数据采集** | 通过 Snap7 协议读写西门子 S7-200/300/400/1200/1500 系列 PLC 的 DB 块数据 |
| **多视图展示** | 工艺流程图、实时数据表、趋势图表、历史趋势四种视图满足不同监控需求 |
| **数据持久化** | SQLite 本地存储 + MySQL 远程同步双保险，支持多粒度数据聚合 |
| **权限管控** | 管理员/工程师/操作员/观察者四级 RBAC 权限体系 |
| **报警管理** | 支持阈值报警配置，多级别告警（信息/警告/错误/严重） |
| **数据导出** | 支持实时数据和历史数据导出为 CSV 格式 |

### 支持的 PLC 型号

- 西门子 S7-200 / S7-200 SMART
- 西门子 S7-300 / S7-400
- 西门子 S7-1200 / S7-1500
- 兼容 Snap7 v1.4.0 协议的其他设备

## 3. 安装与部署

### 3.1 Windows 安装

1. 解压发布包到目标目录（如 `D:\PLCMonitor`）
2. 确保 `snap7.dll` 位于程序目录或系统 PATH 中
3. 运行 `PLCMonitor.exe`

### 3.2 Linux (Ubuntu) 安装

```bash
# 安装依赖
sudo apt update
sudo apt install libsnap7 qt6-base-dev libqt6sql6-sqlite

# 解压并运行
tar -xzf plcmonitor-linux.tar.gz -C /opt/plcmonitor
cd /opt/plcmonitor
./PLCMonitor
```

## 4. 首次启动与配置

### 4.1 启动程序

首次启动时，程序会在以下位置创建配置和数据目录：

- **配置文件**: `<程序目录>/configs/plc_config.json`
- **数据目录**: `<AppData>/PLCMonitor/` (Windows) 或 `~/.local/share/PLCMonitor/` (Linux)
- **日志文件**: `<数据目录>/plcmonitor.log`

### 4.2 配置 PLC 连接参数

编辑 `configs/plc_config.json` 文件：

```json
{
  "plc": {
    "ip": "192.168.0.11",    // PLC 的 IP 地址
    "rack": 0,               // 机架号 (通常为 0)
    "slot": 1,               // 插槽号 (S7-1200/1500 通常为 0 或 1)
    "port": 102              // S7 协议端口 (通常为 102)
  },
  "connection": {
    "auto_connect": true,         // 启动时自动连接
    "auto_reconnect": true,       // 断线自动重连
    "timeout": 3000,              // 连接超时（毫秒）
    "reconnect_interval": 3000,   // 重连间隔（毫秒）
    "max_reconnect_attempts": 3   // 最大重连次数
  },
  "polling": {
    "auto_start": true,     // 连接后自动开始轮询
    "interval": 1000,       // 轮询间隔（毫秒），范围 100-10000
    "enabled": true
  }
}
```

### 4.3 配置监控标签

在 `configs/plc_config.json` 的 `tags` 数组中添加需要监控的标签：

```json
{
  "tags": [
    {
      "name": "motor_running",     // 标签名称（自定义，用于显示）
      "dbNumber": 1,               // DB 块编号
      "byteOffset": 0,             // 字节偏移
      "bitOffset": 0,              // 位偏移（BOOL 类型用，其他类型填 -1）
      "dataType": "BOOL",          // 数据类型：BOOL/BYTE/WORD/DWORD/INT/DINT/REAL/STRING
      "writable": true             // 是否可写入
    },
    {
      "name": "temperature",
      "dbNumber": 1,
      "byteOffset": 2,
      "bitOffset": -1,
      "dataType": "REAL",
      "writable": false
    }
  ]
}
```

**数据类型说明**:

| 类型 | 字节数 | 说明 |
|------|--------|------|
| BOOL | 1 bit | 布尔值，需指定 bitOffset |
| BYTE | 1 | 8 位无符号整数 |
| WORD | 2 | 16 位无符号整数 |
| DWORD | 4 | 32 位无符号整数 |
| INT | 2 | 16 位有符号整数 |
| DINT | 4 | 32 位有符号整数 |
| REAL | 4 | IEEE 754 单精度浮点数 |
| STRING | 可变 | 字符串 |

### 4.4 首次登录

- 默认管理员账号：**admin**
- 默认密码：**admin123**
- **首次登录后请立即修改密码！**

## 6. 功能模块详解

### 6.1 PLC 连接管理

#### 手动连接

1. 点击顶部工具栏的 **"连接PLC"** 按钮
2. 系统根据配置文件中的 IP/Rack/Slot 参数自动连接
3. 连接成功后状态栏显示 **"PLC: ●已连接"**

#### 自动连接

在 `plc_config.json` 中设置 `"auto_connect": true`，程序启动后 1 秒自动连接 PLC。

#### 断线重连

设置 `"auto_reconnect": true` 后，PLC 连接断开时系统会自动尝试重连（默认每 3 秒重试一次，最多 3 次）。

#### 断开连接

- 通过菜单栏 **连接管理 → 断开PLC连接** 可手动断开
- 断开时需要工程师或管理员权限

#### 状态栏指示灯

| 状态 | 颜色 | 说明 |
|------|------|------|
| 已连接 | 绿色 | PLC 连接正常，数据正在采集 |
| 连接中 | 黄色 | 正在尝试连接 PLC |
| 已断开 | 红色 | 与 PLC 的连接已断开 |
| 轮询中 | 蓝色 | 正在周期性读取 PLC 数据 |

---

### 6.2 工艺总貌

工艺总貌以图形化方式展示生产流程，实时反映各设备的运行状态。

#### 操作步骤

1. 在左侧导航点击 **"工艺总貌"**
2. 在右侧标签管理器中选择要显示的标签（勾选复选框）
3. 点击 **"应用"** 按钮，所选标签的数据将在流程图中实时更新
4. 点击 **"刷新"** 按钮可手动刷新当前显示

#### 功能特点

- 支持图形化流程展示
- 标签数据实时映射到流程节点
- 设备状态变化时图形自动更新颜色/样式

---

### 6.3 实时数据监控

实时数据监控以表格形式展示所有选中标签的当前值，适合需要同时监控大量数据点的场景。

#### 操作步骤

1. 在左侧导航点击 **"实时数据"**
2. 在右侧标签管理器中选择需要监控的标签
3. 点击 **"应用"** 按钮
4. 表格中将实时显示所选标签的值

#### 表格列说明

| 列名 | 说明 |
|------|------|
| 标签名称 | 标签的自定义名称 |
| 地址 | PLC 中的 DB 块地址（如 DB1.DBW2） |
| 数据类型 | 标签的数据类型 |
| 当前值 | 最近一次读取的实时值 |
| 质量 | 数据质量标志（GOOD/BAD/UNCERTAIN） |
| 更新时间 | 最后一次更新的时间戳 |

#### 功能特点

- 支持按标签名称搜索过滤
- 数据变化时对应单元格高亮闪烁
- 可导出当前表格数据为 CSV 文件

---

### 6.4 趋势图表

趋势图表以折线图方式实时展示选中标签的数据变化趋势，帮助分析数据变化规律。

#### 图表控件说明

| 控件 | 功能 |
|------|------|
| **播放/暂停按钮** | 控制图表是否实时滚动更新 |
| **时间范围下拉框** | 选择图表 X 轴时间范围（1/5/10/30/60 分钟） |
| **显示网格** | 切换是否显示图表网格线 |
| **显示图例** | 切换是否显示图表图例 |
| **清除图表** | 清空当前图表上的所有数据 |
| **保存图表** | 将当前图表截图保存为 PNG 图片 |

#### 操作步骤

1. 在左侧导航点击 **"趋势图表"**
2. 在右侧标签管理器选择要绘图的标签
3. 点击 **"应用"** 按钮
4. 图表自动开始绘制折线
5. 使用工具栏按钮调整显示参数

#### 功能特点

- 支持最多 8 条折线同时显示
- 每条折线自动分配不同的颜色
- 图表支持鼠标缩放、拖拽平移
- 数据点自动按时间排序

---

### 6.5 历史数据查询

历史数据查询允许用户查看任意时间段内标签的历史数据趋势。

#### 操作步骤

1. 在左侧导航点击 **"历史数据"**
2. 设置查询时间范围：
   - 使用日期时间选择器选择起始时间和结束时间
   - 默认查询最近 1 小时的数据
3. 点击 **"选择标签"** 按钮，在弹出的对话框中选择要查询的标签
4. 点击 **"查询"** 按钮开始查询
5. 图表中将显示所选标签的历史数据趋势
6. 可点击 **"导出"** 将查询结果导出为 CSV

#### 数据存储说明

- 数据存储于 SQLite 本地数据库
- 启用 MySQL 配置后自动同步至远程数据库
- 系统自动进行分钟、小时、天级别的数据聚合
- 过期数据按策略自动清理

### 6.6 标签管理器

标签管理器位于界面右侧，用于选择需要在当前视图中监控的 PLC 标签。

#### 基本操作

| 操作 | 方法 |
|------|------|
| **选中/取消标签** | 勾选/取消标签前的复选框 |
| **全选** | 点击 **"全选"** 按钮 |
| **清除选择** | 点击 **"清除"** 按钮 |
| **应用选择** | 点击 **"应用"** 按钮将选择同步到当前视图 |
| **搜索标签** | 在搜索框中输入标签名称进行过滤 |
| **刷新标签** | 点击 **"刷新标签"** 按钮从配置文件重新加载标签列表 |

#### 随动功能

标签管理器会自动跟随主视图切换：
- 切换到"工艺总貌"→ 显示工艺标签树
- 切换到"实时数据"→ 显示实时数据标签树
- 切换到"趋势图表"→ 显示图表标签树
- 切换到"历史数据"→ 显示历史数据标签树

每个视图的标签选择是独立保存的，切换视图后会自动恢复上次的选择。

#### 列头说明

| 列 | 说明 |
|----|------|
| 选择 | 勾选框 |
| 标签名 | PLC 标签的自定义名称 |
| 地址 | DB 地址（如 DB1.DBD2） |
| 数据类型 | BOOL/INT/REAL 等 |
| 实时值 | 当前实时值 |
| 描述 | 标签描述信息 |
| 记录DB | 是否记录到数据库 |

### 6.7 控制面板

控制面板用于批量写入 PLC 数据，适合需要设置多个参数的场景。

#### 注意事项

- 仅具有 **操作员** 及以上权限的用户可以写入数据
- 写入的数据类型必须与标签定义的数据类型一致
- 写入操作有重试机制（默认重试 3 次）

### 6.8 报警中心

报警中心用于查看和管理系统报警记录。

#### 报警级别

| 级别 | 颜色 | 说明 |
|------|------|------|
| 信息 | 蓝色 | 一般性通知 |
| 警告 | 黄色 | 需要注意的异常 |
| 错误 | 橙色 | 需要处理的故障 |
| 严重 | 红色 | 紧急情况，需立即处理 |

#### 报警配置

在标签配置中可以为每个标签设置报警阈值：
- 上限值：超过此值时触发报警
- 下限值：低于此值时触发报警
- 报警级别：触发报警的严重程度
- 报警描述：报警时的提示信息

### 6.9 数据写入

#### 单个标签写入

1. 在实时数据表格中，双击可写标签的值单元格
2. 输入新值并确认
3. 系统将值写入 PLC 对应地址

#### 批量标签写入

通过控制面板可批量写入多个标签的值。

## 7. 用户权限管理

系统采用四级 RBAC 权限模型：

| 角色 | 权限等级 | 权限说明 |
|------|----------|----------|
| **管理员** (Admin) | 3 | 所有权限：查看/写入标签、管理标签、管理用户、系统配置 |
| **工程师** (Engineer) | 2 | 查看/写入标签、管理标签、系统配置 |
| **操作员** (Operator) | 1 | 查看/写入标签 |
| **观察者** (Viewer) | 0 | 仅查看数据，不能写入 |

### 用户管理操作（管理员权限）

1. 点击顶部工具栏 **"登录"** 按钮登录
2. 通过菜单 **工具 → 用户管理** 管理用户
3. 可执行：添加用户、修改角色、重置密码、禁用/启用用户

## 8. 数据库配置

### 本地存储（SQLite）

SQLite 数据库默认存储在用户数据目录，无需额外配置，程序启动时自动创建。

### 远程同步（MySQL，可选）

在 `plc_config.json` 中配置 MySQL 参数：

```json
{
  "plc_config": {
    "mysql_host": "192.168.1.100",
    "mysql_port": 3306,
    "mysql_database": "plc_monitor",
    "mysql_user": "plc_user",
    "mysql_password": "your_password"
  }
}
```

配置后，系统将：
1. 所有数据先写入本地 SQLite
2. 定期（约每 30 秒）将本地数据同步至远程 MySQL
3. 通过心跳机制检测 MySQL 连接状态

### 数据聚合策略

| 粒度 | 说明 |
|------|------|
| 原始数据 | 按轮询间隔采样（默认 1 秒） |
| 分钟聚合 | 每分钟最小值/最大值/平均值 |
| 小时聚合 | 每小时（基于分钟数据聚合） |
| 天聚合 | 每天（基于小时数据聚合） |

## 10. 常见问题与故障排除

### 10.1 无法连接 PLC

**可能原因及解决方法：**

| 原因 | 解决方法 |
|------|----------|
| IP 地址不正确 | 检查 `plc_config.json` 中的 `plc.ip` 配置 |
| Rack/Slot 号不对 | S7-300/400 默认 Rack=0, Slot=2；S7-1200/1500 默认 Rack=0, Slot=1 |
| 网络不通 | ping PLC IP 地址确认网络连通 |
| 防火墙阻止 | 确保 PLC 的 102 端口对监控 PC 开放 |
| PLC 未启用 PUT/GET | S7-1200/1500 需在 TIA Portal 中启用"允许来自远程对象的 PUT/GET 通信" |

### 10.2 标签值显示异常

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 值始终为 0 | 地址/偏移量配置错误 | 检查 byteOffset 和 bitOffset |
| 浮点数异常 | REAL 和 INT 字节序混淆 | 确认 dataType 与实际 PLC 类型一致 |
| 质量显示 BAD | 标签地址超出 DB 范围 | 确认 DB 块大小和标签地址范围 |

### 10.3 程序启动失败

**Windows 常见错误：**

| 错误 | 解决方法 |
|------|----------|
| "缺少 MSVCP140.dll" | 安装 Visual C++ 2019+ Redistributable |
| "找不到 snap7.dll" | 将 `snap7.dll` 复制到程序目录 |
| "配置文件损坏" | 删除 `configs/plc_config.json` 后重新启动（自动生成默认配置） |

**Linux 常见错误：**

| 错误 | 解决方法 |
|------|----------|
| "libsnap7.so not found" | `sudo apt install libsnap7` 或将库文件放入 `/usr/lib/` |
| "段错误" | 检查 Qt6 运行库完整性；检查配置文件 JSON 格式 |
| "无法打开日志文件" | 检查用户数据目录的读写权限 |

### 10.4 性能优化建议

- 轮询间隔不要设置过小（建议 ≥ 500ms）
- 趋势图表中选择的标签不宜过多（建议 ≤ 8 个）
- 定期清理历史数据，避免 SQLite 数据库过大
- 启用 MySQL 同步时确保网络稳定
